클래스
: 서로 관련있는 데이터들의 집합으로 데이터와 데이터 뿐만 아니라 메서드까지 포함함
	**데이터와 관련깊은 메서드를 묶을 수 있음 
	ex) 클래스와 관련있는 데이터 혹은 메서드를 클래스에 포함한다면 굳이 argument를 받을 필요없이 멤버변수로서 해결할 수 있음
=> 코드의 응집도를 높이고(관련 있는 것들끼리 모임) 결합도(관련없는 부분끼리 영향을 덜 줌)를 낮춤

But, 결국 클래스는 설계도(뼈대)에 불과하고 각각의 객체들이 실질적인 의미를 갖음


객체
: 데이터의 '정적 속성'과 메서드의 '기능(동적 속성)'을 객체가 가지고 있어 '데이터+메서드'로 표현함 


생성자
: 클래스와 동일한 이름을 가지며 return 타입이 없는 메서드
	**객체 생성시 자동으로 실행됨
	**생성자가 반드시 매개변수를 받지 않아도 됨
		목적
		: 객체의 데이터 필드 초기화

	형식
		public 클래스명(원하는 필드){
			필드1 = ?;	//원하는 필드값을 한번 초기화
		}
		...
		main(){
			클래스명 클래스변수 = new 클래스명(필드1, 필드2...);
		}
			**만약 생성자 인수와 클래스 필드명이 겹치더라도 this.필드를 통해 같은 이름이라도 구별해 사용할 수 있음

**클래스 내에 값을 문자열로 반환할 때는 'toString'이라는 이름 그대로 메서드를 써야함(내용은 다르더라도)

결론: 클래스를 통해 관련있는 데이터 혹은 메서드를 모아둠으로써 응집도를 높여 다른 클래스와의 상호 의존성(결합도)을 낮출 수 있음
