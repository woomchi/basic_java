static 멤버
: 일반적인 클래스 멤버와 달리 클래스 안에 존재하며 객체에는 존재하지 않는 클래스 멤버
	=> 객체에 존재하는 멤버와 달리 클래스 안에서 유일한 존재

static
1.main 메서드
: 자바에서 프로그램은 순수하게 클래스들의 집합인데 클래스는 실체가 없는 설계도임 => 클래스로는 아무것도 할 수 없음
	=> main이 static이어야 프로그램 시작점으로써 프로그램을 작동하도록 할 수 있음
	**static의 여부에 따라 클래스 멤버 / 객체 멤버로 나뉨
	
2. static 메서드 -> non-static 멤버로의 액세스
: static 멤버는 프로그램이 시작됐을 때 존재하고, 객체는 new를 통해 객체를 생성했을 때부터 존재함
	=> 객체를 생성한 시점부터 비로소 non-static 멤버로 액세스가 가능함

3. 다른 클래스에 속한 static 멤버로의 액세스
: static 멤버는 객체가 아닌 클래스 멤버이므로 클래스.static멤버 형태로 사용해야함
	=> static 멤버는 클래스 멤버이고 객체가 아닌 클래스 자체에 존재하기 때문에 값이 바뀐다면 객체에선 모두 동일하게 반영됨

4. static 메서드 / 필드 용도
	1) main 메서드
	2) 상수 혹은 클래스 당 하나만 유지하고 있으면 되는 값(혹은 객체)
		ex) Math.PI, System.out
	3)순수하게 기능만으로 정의되는 메서드
		ex) Math.abs(k), Math.sqrt(n), Math.min(a,b)

=> static 멤버 이해를 위해선 프로그램을 작성할 때 static main메서드를 제외하고는 나머지를 최대한 non-static으로 작성하기
해결 방법
: 기존 static main 메서드 역할을 임의의 메서드를 생성해 그 안에 코드를 넣고 해당 메서드를 main에서 파일 객체를 생성해 파일.메서드를 통해 main만이 static이 존재할 수 있게 만듬


접근 제어
default : 동일 패키지에 있는 다른 클래스에서 접근 가능함
public : 클래스 외부에서 접근이 가능함
private : 클래스 내부에서만 접근이 가능함(다른 클래스에서 접근 X)
protected : 동일 패키지에 있는 다른 클래스와 다른 패지키의 하위 클래스에서도 접근 가능
	ex) 수퍼 클래스의 필드가 private이라 서브 클래스에서 액세스할 수 없을 때 사용 가능

데이터 캡슐화(=information hiding)
: 모든 데이터 멤버를 private으로 만들고 필요에 따라 public한 get/set 메서드를 제공하는 것
	=> 객체가 제공하는 메서드를 통하지 않고선 객체 내부 데이터에 접근할 수 없음
		ex) private int coef; => public int getCoef() {   return coef;   }
			**이때 private 필드의 값을 읽을 수는 있지만, 쓸 수는 없음
			**필드를 refactor-encapsulate filed를 통해 get/set 함수를 자동으로 생성할 수 있음
